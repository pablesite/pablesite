# Planning Guide

Usar esta guía cuando el usuario quiera planificar trabajo antes de implementarlo.

## Step 1 — Read before speaking

1. `docs/project-status.md`
2. La documentación del área afectada
3. Las task specs existentes en `docs/tasks/` si las hay

## Step 2 — Present interpretation

Resumir:

1. qué parece pendiente
2. qué ya existe
3. cómo se puede partir en fases o tareas
4. qué ambigüedades hay

## Step 3 — Incorporate user input

Esperar la respuesta del usuario antes de generar specs.

## Step 4 — Generate task specs

Crear archivos bajo:

```text
docs/tasks/<module>/phase-N-<short-name>/
```

## Step 5 — Update project status

Añadir las nuevas tareas a `docs/project-status.md`.
