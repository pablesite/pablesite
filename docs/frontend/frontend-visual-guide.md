# Frontend Visual Guide

## Objetivo

Evitar que la UI derive hacia soluciones improvisadas y difíciles de mantener.

## Principios

1. Una jerarquía visual clara por pantalla
2. Espaciado consistente
3. Un patrón reusable antes que estilos locales repetidos
4. Estados explícitos: loading, empty, error, success
5. Copy corto y orientado a la acción

## Reglas

1. Evitar `style=` inline
2. Evitar hardcodes repetidos de spacing, width y radius
3. Promover clases compartidas y tokens antes de variantes locales
