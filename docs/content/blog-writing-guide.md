# Blog Writing Guide

Guía canónica para escribir artículos del blog de `pablesite`.

## Objetivo

Mantener una estructura reconocible, con voz personal y suficiente densidad
técnica para que cada artículo sea útil aunque el tema nazca de una experiencia
propia.

## Estructura obligatoria

Todos los posts deben seguir este orden:

1. `TL;DR`
2. `Motivación`
3. `Solución aplicada`
4. `Reflexiones`

## Qué va en cada sección

### `TL;DR`

1. Debe ir al principio del artículo
2. Debe ser breve
3. Debe resumir de 3 a 5 ideas clave
4. Debe permitir entender el valor del artículo sin leerlo completo

### `Motivación`

1. Explica el contexto personal, el problema o la necesidad real
2. Prioriza honestidad y concreción sobre storytelling artificial
3. Si el origen del artículo fue una incomodidad operativa, económica o de
   producto, dejarlo explícito

### `Solución aplicada`

1. Explica qué hiciste de verdad
2. Incluye decisiones, tradeoffs, herramientas, arquitectura o pasos técnicos
   cuando aporten valor
3. Evita tutoriales inflados: mejor contar la solución concreta que intentar
   enseñar todo el dominio

### `Reflexiones`

1. Cierra con aprendizaje, límites, cosas que cambiarías o conclusiones útiles
2. No repetir el TL;DR
3. Debe sentirse como cierre personal, no como texto corporativo

## Tono editorial

1. Voz personal, directa y sobria
2. Técnica cuando hace falta, pero sin postureo
3. Nada de hype, autocelebración vacía ni copy inflado
4. Mejor ejemplos concretos que abstracciones largas
5. Si una decisión fue imperfecta, decirlo

## Flujo recomendado con agentes

Cuando uses un agente para redactar:

1. Tú aportas las ideas clave y el contexto real
2. El agente propone o redacta el artículo respetando la estructura obligatoria
3. El agente debe sonar cercano a tu tono y evitar giros genéricos de IA
4. Si faltan datos, el agente debe dejar huecos claros o pedir solo la mínima
   información necesaria

## Checklist rápido antes de publicar

1. ¿Hay un `TL;DR` con 3-5 ideas útiles?
2. ¿La motivación cuenta por qué esto importaba de verdad?
3. ¿La solución aplicada describe lo que realmente hiciste?
4. ¿El cierre aporta reflexión en vez de repetir el contenido?
5. ¿El texto suena a Pablo y no a plantilla impersonal?
